Title  [TC2] Receive a valid purchase order
 Description  Test case that sends a valid purchase order to the SUT and expects a correct response.

 

Test step notes

  • You will receive a purchase order at your API endpoint (defined as part of your system's configuration).
  • You need to respond with a 200 (OK) response and a text/plain body.
  • The content of the body must be a reference code starting with "REF-" and followed by one or mode digits.

The project also provides an online validator as a complementary service.
